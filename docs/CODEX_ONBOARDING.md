# Codex導入・理解確認手順 v0.1

## 目的

Codexに最初から実装をさせない。

最初に、プロジェクトの目的・選択肢の原則・変更権限・実装上の優先事項を正しく読めるかを確認する。

この手順を通過するまで、ゲーム本体の実装作業には進まない。

## 1. 実行前の準備

Codexに接続するGitHubリポジトリのルートに、少なくとも以下があることを確認する。

```text
AGENTS.md
docs/GAME_DESIGN.md
docs/CHOICE_RULES.md
docs/PRODUCTION_WORKFLOW.md
```

## 2. 最初にCodexへ送るプロンプト

以下をそのまま貼り付ける。

```text
このリポジトリの以下の資料を、最初にすべて読んでください。

- AGENTS.md
- docs/GAME_DESIGN.md
- docs/CHOICE_RULES.md
- docs/PRODUCTION_WORKFLOW.md

この作業では、コード・文書・設定を一切変更しないでください。

読了後、次の5点だけを日本語で報告してください。

1. このゲームの教育的な目的を、3〜5文で説明してください。
2. 選択肢の設計で、特に避けるべき表現・構造を3点挙げてください。
3. Codexが、教員または設計者の確認なしに変更してはいけないものを挙げてください。
4. 試作版で実装上、優先すべきことを優先順位付きで示してください。
5. 資料間で曖昧、または判断が必要だと考える点を挙げてください。

最後に、今回読み込んだ資料名を箇条書きで再掲してください。
```

## 3. 教員・設計者が確認する基準

### 合格とみなせる回答

- 目的を「SNSの危険防止」だけに単純化していない。
- 「民主社会における意見形成」「異なる意見との関係」「情報環境への自覚」に触れている。
- 選択肢の善悪を露骨に分けない原則を理解している。
- シナリオ・人物設定・選択肢・結末を勝手に変えないことを明記している。
- iPad操作性、個人情報を扱わないこと、既存分岐を壊さないことに触れている。
- 曖昧な点を、勝手に補完せず質問・懸念として挙げている。

### 保留・差し戻しとする回答

次のいずれかがあれば、実装を依頼せず、資料またはプロンプトを改善する。

- 「正しい情報を選ばせるゲーム」と説明している。
- 「慎重に事実確認する」を常に正解として扱っている。
- 動画投稿者・親友・学校側の誰かを明確な悪役として理解している。
- 生徒の選択を点数化・診断する前提で提案している。
- 資料にない社会問題・人物設定を勝手に追加しようとしている。

## 4. 合格後の最初の実装タスク

理解確認に問題がなければ、次に画面の骨組みだけを依頼する。

```text
AGENTS.md と以下の資料を読んでから作業してください。

- docs/GAME_DESIGN.md
- docs/CHOICE_RULES.md
- docs/PRODUCTION_WORKFLOW.md

今回は、シナリオ本文・選択肢・人物設定・結末の意味を変更してはいけません。

会話画面、選択肢表示、SNSタイムライン画面、グループチャット画面、最初からやり直すボタンの骨組みだけを実装してください。

完成条件：
- iPadのブラウザで文字とボタンが読みやすい。
- 選択肢を押すと次の仮画面へ遷移できる。
- 画面が大きく崩れない。
- リセットで初期画面に戻る。
- 外部ライブラリ、ログイン、個人情報の送信・保存を追加しない。

作業後は、AGENTS.md の「作業終了時の必須報告」の形式で報告してください。
```

## 5. 今後の運用

- 新しい章・大きな分岐・人物追加の前には、必ず `docs/specs/` に仕様書を作る。
- Codexへの依頼では、読むべき資料・変更してよい範囲・完成条件・報告形式を毎回明示する。
- Codexの提案が教育設計に関わる場合、実装前にChatGPTと教員でレビューする。
- 同じファイルを複数のCodexスレッドで同時に編集させない。
