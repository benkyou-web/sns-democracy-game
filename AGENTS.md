# SNSノベルゲーム：Codex作業指示

このファイルは、このリポジトリで作業するCodex向けの最優先ルールです。
実装・修正・レビューの前に必ず読み、以下の指示を守ってください。

## 1. 作業開始前に必ず読む資料

作業を始める前に、次の資料を確認してください。

1. `AGENTS.md`
2. `docs/GAME_DESIGN.md`
3. `docs/CHOICE_RULES.md`
4. 今回の作業に関係する仕様書（例：`docs/specs/chapter01.md`）
5. 既存のHTML、CSS、JavaScript、データファイル

資料間に矛盾や不足がある場合、教育的な内容を勝手に補わず、作業報告の「懸念点」に明記してください。

## 2. このプロジェクトの目的

このゲームは、正しい選択肢を選ばせるための教材ではありません。

生徒が物語の選択を重ねながら、次の点を振り返るための高校「公共」の教材です。

- 自分がどのような情報に触れてきたか
- 自分が誰の意見を「普通」「正しい」と感じるようになったか
- フィルターバブルやエコーチェンバーによって、何が見えにくくなるか
- 意見の異なる人と、どのように関係をつくり、社会の問題を考えるか

最終目標は「SNSに気をつけよう」と教えることではなく、異なる意見がある民主社会で、根拠・背景・他者との関係を考えながら判断する姿勢を促すことです。

## 3. 選択肢・人物表現に関する絶対条件

### 選択肢

- 選択肢を、明白な善悪・正誤の形にしない。
- 各選択肢には、守れるものと、見落とす可能性の両方を持たせる。
- 「冷静に事実確認する」「感情的に拡散する」のような、教材の正解が透けて見える表現を使わない。
- 選択の結果を、その場で「正しい」「間違い」と断定しない。
- 既存の選択肢・台詞・分岐の教育的な意味を、指示なく変更しない。

### 人物

- 特定の人物や立場を、単純な悪役・愚かな人物として扱わない。
- すべての主要人物に、守ろうとしているもの、事情、見落としやすい点がある前提を守る。
- 実在の生徒、学校、教職員、政治家、政治団体、事件を直接モデルにしない。

## 4. 実装上の優先事項

1. iPadのブラウザで読みやすく、押しやすく、正常に動くこと
2. 生徒の個人情報を取得・送信・保存しないこと
3. 既存の分岐、選択履歴、リセット機能を壊さないこと
4. 会話画面、SNSタイムライン、チャット、コメント欄などの画面が区別しやすいこと
5. 外部サービスや外部APIに依存せず、静的サイトとして動作できること
6. 変更を小さく分け、既存の設計を不用意に大きく変更しないこと

## 5. 変更してよい範囲と、確認が必要な範囲

### 事前確認なしで変更してよいこと

- 明らかな表示崩れ・操作不能・構文エラーの修正
- 仕様書に明記された画面遷移・UI・データ処理の実装
- iPad表示のためのレイアウト調整
- 指示された文言の反映

### 教員または設計者の確認が必要なこと

- シナリオ本文、選択肢、人物の価値観、結末の意味を変更すること
- 新しい登場人物・事件・政治的論点・社会的事例を追加すること
- フィルターバブル、エコーチェンバー、民主主義に関する説明を追加・変更すること
- 生徒の行動を評価・診断するような表現やスコアを追加すること
- 外部ライブラリ、ログイン、データ送信、アクセス解析を追加すること

## 6. ブランチと変更管理

- `main` は授業配布用の安定版です。原則として直接変更しないでください。
- 新機能は `feature/機能名`、修正は `fix/修正名` を使用してください。
- 作業前に、既存の変更内容を確認してください。
- 変更は必要最小限にし、関係ないファイルを大規模に整形・書き換えないでください。

## 7. 作業終了時の必須報告

作業が終わったら、必ず次の形式で報告してください。

1. **変更したファイル**
2. **実装・修正した内容**
3. **実施したテスト**
4. **残っている懸念点・未確認事項**
5. **教員または設計者が確認すべき点**

報告では、教育的な内容を変更した場合、その箇所と理由を必ず明示してください。
