# 意思決定ログ v0.1

重要な設計変更を記録し、後から「なぜこの形にしたか」を確認できるようにする。

## D-001：制作形式

- **決定**：HTML/CSS/JavaScriptによる静的Webゲームとして作る。
- **理由**：生徒のiPadからURLで開きやすく、SNS風の画面を作りやすい。インストールを不要にできる。
- **日付**：2026-07-04

## D-002：公開方法

- **決定**：GitHub Pages等の静的ホスティングでURL配布する。
- **理由**：Classroomで配布しやすく、更新版を同じURLで提供できる。
- **日付**：2026-07-04

## D-003：個人情報・アクセス解析

- **決定**：生徒の氏名、メールアドレス、回答、アクセス履歴を収集・送信しない。
- **理由**：教材の公開・配布を安全かつ簡潔に行うため。
- **日付**：2026-07-04

## D-004：選択肢の基本原則

- **決定**：明白な正解・不正解を設けず、各選択肢に守れる価値と見落とし得る点を持たせる。
- **理由**：道徳テスト化を避け、情報環境・人間関係・民主的対話の難しさを体験させるため。
- **日付**：2026-07-04

## D-005：試作版の規模

- **決定**：第1章のみ、15〜20分、三つの選択、二〜三の振り返りに絞る。
- **理由**：本番に必要な分岐・画面・個別振り返り・iPad操作を、早い段階で検証するため。
- **日付**：2026-07-04
