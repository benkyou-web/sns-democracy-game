# SNSノベルゲーム制作・運用規約 v0.1
## ChatGPT × Codex × GitHub による試作版制作フロー

## 1. 基本方針

このゲームは、正しい選択肢を選ばせる教材ではない。

生徒が選択を重ねるなかで、次のことを振り返るための高校「公共」の教材とする。

- 自分が何を見てきたか
- 誰の意見を「普通」「正しい」と感じるようになったか
- 何を見落としていたか
- 異なる意見の人と、どのように関係をつくるか

試作版は規模を絞るが、制作の手順・確認方法・公開方法は本番と同じにする。

## 2. 役割分担

| 担当 | 主な役割 |
|---|---|
| 教員 | 教材責任者。教育目標、価値の扱い、人物表現、公開可否を最終判断する。 |
| ChatGPT | 企画、シナリオ編集、選択肢設計、教育的レビュー、仕様書作成を担う。 |
| Codex | HTML/CSS/JavaScript実装、動作確認、バグ修正、コード上の改善を担う。 |
| GitHub | コード・資料・変更履歴・公開版を管理する。 |

### 原則

- ChatGPTとCodexに、同じ仕事を重複して任せない。
- ChatGPTは「何を作るか」「なぜ作るか」を明確にする。
- Codexは「どう動かすか」「どう壊れにくくするか」を担当する。
- 教員は「授業として成立するか」を最終判断する。

## 3. 試作版の範囲

### 目的

本番の縮小版ではなく、**本番に必要な仕組みを一通り検証する縦切りの試作**を作る。

### 想定

- 舞台：学校祭のルール変更をめぐる動画拡散
- プレイ時間：15〜20分
- 登場人物：主人公、親友、実行委員、担任または顧問、動画投稿者
- 選択場面：3回
- 画面：会話、タイムライン、グループチャット、DM、コメント欄、振り返り
- 結末：2〜3種類

### 必須機能

1. 選択により、次に表示される投稿が変わる。
2. 選択により、人物の返答や相談相手が変わる。
3. 選択により、最後の振り返りの問いが変わる。
4. 選択肢に露骨な善悪をつけない。
5. iPadのブラウザで無理なく操作できる。
6. 最初からやり直せる。
7. 生徒の個人情報を取得・送信・保存しない。

## 4. 制作サイクル

### 4-1. ChatGPTで仕様を作る

教員の意図をもとに、ChatGPTが「仕様カード」を作る。

仕様カードには、少なくとも以下を含める。

- 機能名
- 教育的な狙い
- プレイヤーに起きること
- 選択肢
- 各選択で守れるもの
- 各選択で見落とし得るもの
- 画面・人物・内部状態の変化
- 完成条件

### 4-2. Codexに実装を依頼する

Codexには、対象の仕様書と読み込むべき資料を明示して依頼する。

- 依頼前に `AGENTS.md` と関連する `docs/` を読むよう指定する。
- 教育的な内容・人物の価値観・選択肢を勝手に変更しないよう明記する。
- 作業範囲を小さく限定する。
- 完成条件と報告形式を明示する。

### 4-3. Codexの結果を確認する

Codexの報告は、必ず以下の形式にする。

1. 変更したファイル
2. 実装・修正した内容
3. 実施したテスト
4. 残っている懸念点・未確認事項
5. 教員または設計者が確認すべき点

### 4-4. ChatGPTで教育的レビューを行う

次の観点でレビューする。

- 選択肢が露骨に誘導的ではないか。
- 特定の人物を単純な悪役にしていないか。
- SNSの問題を個人の性格の問題に矮小化していないか。
- フィルターバブルやエコーチェンバーが、体験として表現されているか。
- 民主主義・対話・人間関係への問いにつながっているか。

### 4-5. 教員が端末で確認する

教員は、コードではなく次を確認する。

- 文章が高校生にとって自然か。
- ボタンや文字がiPadで操作・閲覧しやすいか。
- 選択肢に違和感や誘導感がないか。
- 生徒が誤解しそうな表現がないか。
- 授業時間内で進められそうか。

## 5. GitHub運用

- `main`：授業配布用の安定版
- `feature/機能名`：新機能・新章の作業用
- `fix/修正名`：不具合修正用

原則として、Codexは `main` を直接変更しない。

変更は作業用ブランチで行い、変更内容を確認してから `main` に取り込む。

## 6. 試作版の完成条件

試作版は、次を満たした時点で完成とする。

- 15〜20分で一周できる。
- 三つの選択があり、選択により見えるSNS投稿が変わる。
- 少なくとも二つの異なる結末または振り返りがある。
- 生徒ごとに異なる振り返りの問いが表示される。
- プレイ中は、フィルターバブルやエコーチェンバーを用語として説明しない。
- プレイ後の比較活動で概念に気づける。
- iPadで画面崩れ・操作不能がない。
- 実在の個人・学校・政治団体・事件・生徒情報を扱わない。

## 7. 本番版へ進む判断基準

試作版の後、次の問いにすべて「はい」と答えられる場合に第2章以降へ進む。

- 生徒が「正解探し」ではなく、自分の選択の意味を考えられたか。
- 異なるプレイ結果を比較することで、情報環境の違いに気づけたか。
- 選択肢が露骨に誘導的ではなかったか。
- iPadで無理なく操作できたか。
- 授業後の振り返りが民主主義・対話・人間関係につながったか。
